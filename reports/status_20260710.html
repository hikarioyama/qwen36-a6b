05mechanism の現在の絵

今夜の実測を 1 枚に畳むと:

  • fact は k8 経路に無傷で残っている(joint@k8 0.8533)。k32 の知識劣化は「知識が消えた」のではなく「tail 24 expert の未較正出力が fact の logit margin を薄めて読めなくしている」。
  • だから (a) 推論時に tail 質量を減衰しても (b) 訓練で k32 分布を k8 分布に合わせても、どちらでも回復する — 2 本のレバーが同じ壁の別の面を叩いている。
  • math は逆に tail expert の実計算を使っており(damp で −3.7pt / 訓練で 0.890 完治)、knowledge と math で最適 gate 形状が違う。
  • 「重み容量が足りない」説を支持する実測は現存しない — full-FFN の根拠は崩れた。

06integrity 事故 2 件と修復(正直な記録)

事故①: 「stock base」と思っていた local モデルが merged モデルだった。 /mnt/data/models/Qwen3.6-35B-A3B-agentic-k32 は README が stock のままだが、重み指紋(agentic 選抜内の expert)が stock と不一致 = agentic patch 焼き込み済みモデル。今日日中の joint probe eval 群はこれを base にした「キメラ」測定だった(patch が選抜 833 expert + router を上書きするため汚染は選抜外のみ、フラット性の結論は不変、絶対値のみ 0.8133→0.8183 に修正)。

事故②: aux-host の eval ハーネスが choice-logprob 未実装の旧版だった。 生成ベース MMLU で走った初回 G0 sweep は k32 0.8517 > k8 0.805 という物理と逆の値を出した(無効、退避済み)。

検出の経緯が教訓そのもの: aux-host の「あり得ない逆転」を配線バグと疑い、3 ノードの重み指紋照合で①が、tok/s>0(logprob なら 0)で②が確定。モデル dir は README や名前でなく重み指紋で検証する。物理に反する数字は必ず配線を疑う。修復: 真 stock で全対照を再アンカー(§02)、aux-host へ新ハーネス移植、以後の数字はすべて修復後。

その他の踏み抜き(修復済): ①router 永続化バグ(checkpoint に訓練済み router が保存されず初回 probe 1200 step が無評価に → save/load 修正・save-test で 120 キー実証)②FSDP×Adafactor の optimizer state 保存が scalar RMS の rank 間 assert で必ず死ぬ(model-only checkpoint で回避)③FSDP use_orig_params の grad は rank の shard 所有分のみ(union 判定に修正)④HumanEval の default max_new で 55% truncation(4096 で再走中)。

# α ダイヤル (router tail-scale) — 参照文書

定義: 各 MoE 層の top-32 重みのうち、トークンごとの順位 9-32 位を α 倍 → 合計 1 に再正規化する実行時操作。重み不変。
根拠となる構造: top-k は入れ子 (k32 の上位 8 = k8 の 8) → k 拡張で壊れるのは選択でなく配合比のみ。α=0 は k8 の再正規化と数学的同一。

## 数値例 (1 トークン)
上位 8 人合計 18% (1 位 A=6%)、rank 9-32 合計 20% のとき:
- k=8: A の取り分 6/18=33.3%
- k=32 素: 6/38=15.8% (上位 8 人合計 47%)
- k=32+α0.5: 6/28=21.4% (上位 8 人合計 64%)
- k=32+α0: 33.3% = k8 と一致

## 実装
- eval: esft/eval_harness.py `--router-tail-scale` (gate forward hook: argsort 降順 → 位置 8+ に α → float32 renorm)
- trainer: train_fullffn_dcp.py 同名フラグ (train + in-training eval + checkpoint 再計算、α を checkpoint メタデータに記録、resume 時照合)
- 正規化後の scores への適用は正規化前と可換 (同比率同和)。

## 検証
α=0 vs base@k8: 予測一致 98/100 (bf16 ノイズ水準。同一重みのマシン間一致 98.7% と同等)。

## 実測 (MMLU n=600 同一 items, paired, 2026-07-12)
base:      α=0: 84.67 / 0.25: 84.33 / 0.5: 84.50 / 0.75: 82.83 / 1.0: 81.50 — [0,0.5] 平坦→崩壊。借金 −3.17pt は α≤0.5 で全額返済
joint1000: α=0: 83.17 / 0.25: 84.00 / 0.5: 82.83 / 0.75: 82.17 / 1.0: 81.00 — 最良でも床未満、最適 α が低下 (router 平坦化と整合)

## 役割
①較正の肩代わり (router 凍結を可能に) ②ダイヤルネイティブ訓練 (訓練と推論の配合一致) ③境界の掃引 = 進捗メーター (最適 α の上昇 = k32 容量の解放) ④推論サーバ設定として出荷可

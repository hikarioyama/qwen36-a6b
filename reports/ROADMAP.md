- **借金完済**: base@k8 84.67 vs tail05@k32+α0.5 **84.83** (MMLU n=600 paired, ns) — キャンペーン初の床超え点推定。JMMLU −0.67 ns / GSM8K +2.75 ns で 3 軸無害、k8 非劣化 PASS (−0.50 ns)
- **α ダイヤルの新規性**: 手法は既出調査で未発見 (現象 = k 増しの壁は Elastic MoE arXiv:2509.21892 が先行)。詳細は reports/PRIOR_ART_ALPHA_DIAL.md
- **v4 の訓練傾きはゼロ** (3 軸、同一 α paired) — 燃料の質が主容疑者
- **能力の崖は T3→T4 間**: 意図レベル selfgen の tier 別採用率 T1 97% / T2 87% / T3 90% / **T4 26-28% (3 run 再現)**。T4 gold (機械検証済み成功 trace) 在庫 2,457 行
- **動的 K は死亡**: per-token keff95 は 29-30/32 に単峰集中、easy-token 部分集団 0.0% (n=21M 観測)
- 汚染: intent 全 3 ラウンド 7,388 行、3 段階監査でゼロヒット
- **思考長プローブ陽性**: 同一問題にゼロ思考正解と長思考正解が共存する seed が 80% (151/188、T2/T3、n=1,600 rollouts) — 判定基準 (2 割) を大幅クリア。正解の 58.7% はほぼ思考ゼロ、思考時 p50=3,617 chars の二峰分布
- **【重大】v4 訓練は BFCL を −31pt 壊していた**: base@k8 84/100 vs tail05@k8 53/100 (paired, McNemar p=1.2e-07)。ダイヤル・k32 は無罪 (切り分け済み)。**機構 = 関数名の複写忠実度の喪失** (mock_* テンプレ名への過適合で名前を「再構成」する癖; call 数・引数値は無傷)。v5/v5.1 も同燃料構成

## RL 戦線 (ThinkingCap 路線)

目標: 「思考オン時に短く考える」を verifiable reward (機械検証の合否 − λ×思考長) の GRPO で獲得。蒸留 (27B teacher) は不採用 — teacher が天井になるため。

1. プローブ → INC-0 → GRPO の三段ゲート (各段が安価に前段を殺せる)
2. 設計資産: esft/rl/RL_DESIGN.md (旧 delta 線用)。適合 2 点 = 訓練対象を full-FFN に / 報酬を機械検証+思考長に。当時のブロッカー (vLLM serve 故障) は解消済み
3. 実行配置: rollout = ローカル vLLM lane / 訓練 = gpu-host の区間の合間
4. **budget forcing (thinking トークン上限 + 強制クローズ) は訓練と独立に出荷ノブとして標準装備**

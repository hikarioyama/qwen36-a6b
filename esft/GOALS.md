# ESFT for Qwen3.6-35B-A3B — Goal 定義 (2026-07-03)

**一行**: top-k を K* に上げ、各ドメインの担当 expert を full FT し、**明確な(CI で分離した)性能アップ**を達成、
**NVFP4 に高精度で焼いて** serve できる状態にする。LoRA なし・router 凍結。

## 0. 測定の基準線(4段の比較チェーン)

各ドメインで必ずこの4点を同条件で測る。これがないと「どのレバーが効いたか」が分離できない。

| ラベル | 意味 | 役割 |
|---|---|---|
| **base@8** | native、無訓練 | 正直な floor |
| **base@K\*** | top-k↑、無訓練 | top-k 拡張"だけ"の寄与(=タダのレバー) |
| **ESFT@K\*** | top-k↑ + 担当 expert FT | **これが製品** |
| **27B-dense** | Qwen3.6-27B(quality 兄弟) | 憧れのバー |
| **ESFT@K\*-NVFP4** | 上を NVFP4 に焼いた版 | **出荷物**。BF16 版との差が精度税 |

## 0.5 ハード制約: active ≤ 9B (A9B 天井、ユーザ指定 2026-07-03)

active(K) = 2.48B + K×0.129B → **K ≤ 50** で active ≤ 9B(K=50:8.93B, K=51:9.06B)。
K* はこの天井を絶対に超えない。sweep 候補は全部下(k24=5.6B, k32=6.6B)なので現状は非拘束だが、
今後 K を上げる実験でも **K=50 を上限**とする。

## 0.7 ミッション: 35B-A3B を限界まで強く(ユーザ 2026-07-03)

27B は参考バーに格下げ。**主目的 = 35B-A3B の限界性能**。sweep が「k=8 は潜在能力を出せてない」を実証
(無訓練 top-k↑ で +12.7pt)。伸びしろを全部引き出す。

**honest な成功バー(引き上げ)**: 主判定を `ESFT@K* > base@8` から **`ESFT@K* > base@K*`(=base@32)** に強化。
base@32=0.850 が既にタダのレバーで出てる以上、「訓練した意味」= その上に CI 分離で積むこと。ゆるいバーで満足しない。

**限界レバースタック(大きい順・honest EV)**:
1. top-k=32(確定 +12.7pt、A9B 天井=ピーク)— 引き済み
2. ESFT@32 選抜 expert 訓練(EMoE +0.6〜2.7 / ESFT論文 +数〜15pt)— G2 の本命
3. 選抜 p 0.2→0.3(容量↑・過学習リスク、膝を探す)
4. 訓練トークン量を step 500 超へ(held-out で膝)
5. router 訓練 ablation(non_expert_modules、汎用毀損リスク大・慎重)
主戦=1+2、3-5 は 2 が効いた後の上積み。全部 A9B + ドメイン非毀損ゲート下。

## 1. 「明確な性能アップ」の定義(数字で)

**主判定**: `ESFT@K* > base@8` が **95%CI で分離**(重複ゼロ)。
- CI を締めるため最終評価は **n≥600**(sweep の n=300 は探索用。n=600 で acc0.77 の CI≈±3.4pt、n=1000 で ±2.6pt)。
- 「トレンド」でなく「決着」を出す。small/負も正直に記録。

**副判定(必須)**: 汎用非劣化。`ESFT@K*` の汎用ベンチ(MMLU/JMMLU 一般部)が `base@8` から **−1pt 以内**。
ESFT の売りは「専門を上げて汎用を壊さない」なので、これを割ったら ESFT の意味が消える。

**stretch(狙うが必達でない)**: `ESFT@K* ≥ 27B-dense`。9× active の dense 兄弟を跨ぐ。
届かなくても主判定を満たせば勝ち。honest な期待値: 全ドメインで跨げるとは限らない、ドメイン差が出る。

## 2. ドメイン別 目標(実データ埋め込み済み=math)

| ドメイン | 主指標 | base@8 実測 | 目標(主判定) | stretch |
|---|---|---|---|---|
| **math** | GSM8K acc | **0.723** (n300) | ESFT@K* が 0.723+ で CI 分離 | ≥ 27B-dense |
| coding | HumanEval pass@1 | 未測 | 同上 | ≥ 27B-dense |
| 日本語 | JMMLU / 生成judge | 未測 | 同上 + 汎用非劣化 | ≥ 27B-dense |
| ツールコール | BFCL AST acc | 未測 | 同上 | ≥ 27B-dense |

## 3. NVFP4 焼き込みの目標(高精度要件)

**焼く対象**: 訓練した選択 expert のみ NVFP4 化(残りは元の量子化を維持)。base は元々 fp4 native。

**精度税ゲート**: `ESFT@K*-NVFP4` の acc が `ESFT@K*-BF16` の **99% 以上**(= 相対劣化 ≤1%、
かつ絶対で主判定=base@8 超えの CI 分離を維持)。
- 焼いて base@8 を割ったら焼き方が失敗(scale 破壊等)。**cosine でなく実ベンチの exact-match で判定**(君の教訓)。
- 検証は空 KV/クリーン状態で(FlashInfer warm cache が bug を隠す教訓)。

## 3.5 偏り防止 + 優先ドメイン(ユーザ指定 2026-07-03)

**優先度**: ツールコール > コーディング > 一貫性 を特に重視。math は「パイプライン実証用の先行」で、
効いたら **coding → toolcall を優先展開**(日本語は後)。

**偏り防止(3層)**:
1. **cross-domain 回帰行列**(必須・新設): あるドメインの patch を焼いたら、そのドメインだけでなく
   **全ドメインのベンチで測る**。patch A が他ドメインを −1pt 超で毀損したら「偏った」= 不合格。
   overlap は 3-10%(実測)で低いので原理的には両立するはず、を数字で保証する。
2. **汎用非劣化ゲート**(既存): MMLU/JMMLU 一般部 −1pt 以内。
3. **eval-best checkpoint**(既存): ≤500 step, 100 step ごと eval, best 採用で過学習を抑制。

**一貫性(consistency)の定義 = 測る**:
- **ツールコール**: (a) format-validity 率(tools schema に合う JSON を出せるか)(b) BFCL AST 一致
  (c) **over-trigger 防止**: 呼ぶべきでない時に呼ばない。→ ⚠️ xLAM は全部 positive tool-call。
  **no-tool-needed の負例を訓練データに混ぜる**(混ぜないと「常にツールを呼ぶ」偏りが焼き付く)。
- **コーディング**: pass@1 + 再現性(greedy で決定的、複数 seed で分散小)。
- 汎用: 同一入力で崩れない(温度0で安定)。

## 4. 実行の階段(取り組みやすい単位に分解)

```
G0  [完了] データ4ドメイン準備 + 集中度ゲート白 + top-k sweep
     └ 成果: routing 集中 math4.2%/code5.4%/ja1.8%、ドメイン重複3-10%、naive曲線
G1  [次] K* 確定 + 27B-dense 現ギャップ実測(4点チェーンの base 側を全部埋める)
     └ ゲート: naive 曲線の膝 = K*。速度税と acc 伸びの交点で決める
G2  math で ESFT@K* 訓練 → n600 eval → 主判定 pass/fail
     └ ★ここが山。手法が実モデルで効くかの決着。EMoE アンカー +0.6〜2.7、ESFT論文 +数〜15pt
     └ kill: base@K* から有意に伸びない(loss 動かない/router 質量が rank9-K*に無い)→ K* 見直し or ESFT@8
G3  math を NVFP4 に焼く → 精度税ゲート(≤1%)→ serve smoke
     └ ★出荷物の完成形を1ドメインで先に通す(量子化の罠を早期に潰す)
G4  coding / 日本語 / ツールコール に展開(G2-G3 の型を反復)
G5  仕上げ: パッチ swap 運用 + 重ね掛け検証(日本語×coding 等)+ 4ドメイン NVFP4 serve
```

**設計思想**: G2+G3 を math で先に貫通させ、「訓練が効く」と「NVFP4 に焼ける」の2大リスクを1ドメインで殺す。
そこを越えれば G4 は同型反復。各 G は独立に pass/fail が付き、進捗が数字で見える。

## 5. 正直な EV(盛らない)

- naive top-k は 256-expert では素で +4.3pt(k=16, ただし n300 では CI 重複=トレンド)。文献予想(フラット)を超える追い風。
- ESFT の主効果は「補欠を鍛える」ことで生む。K* を大きくしすぎると鍛える対象と速度税が増えるだけ → 膝で止める。
- 最大の未知: NVFP4 焼き後の精度保持(G3)。MoE の fp4 焼きは scale 破壊の前科多数(DSv4 MTP の教訓)→ G3 を早期ゲートに。

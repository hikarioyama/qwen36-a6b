Full-FFN パイプライン

35B-A6B を ツールコール・コーディング・一貫性(指示追従)・日本語の4軸で base から有意強化する Full-FFN 訓練パイプライン。データ生成(ローカル)→ 訓練(gpu-host 8×96GB)→ 評価(ローカル)→ 判定ゲート → 反復ループの全体像と、通過済み(✓)/直前(点線)のゲートを示す。

究極目標: 4軸で base から「有意」(同一問題・同一 seed の paired、CI95 が 0 を跨がない) 強化。一般能力 (MMLU / GSM8K) は非劣化ゲートで守る。マシン分担: ローカル=2×RTX PRO 6000(生成工場+eval 計測室)／ gpu-host=8×96GB(訓練炉)／ aux-host=停止中。

  • だが戦略上は死んでいない: verifiable な日本語指示追従の外部データはほぼ存在しない(検証済み)ため、この軸は自己生成が主力。

訓練層

gpu-host 8×96GB Blackwell — Full-FFN 本走炉

32.2Btrainable (32,212,254,720)
k = 32active experts
+8.9%決定論 env コスト
  • 手法: Full-FFN。expert grad union 80/80(frozen grad なし、実測)。
  • router 可動 joint (二重 opt-in): --method full-ffn --train-router --allow-router-joint-fullffn。単独 flag は拒否、full-ffn 以外との併用も拒否。
  • router LR = router_lr_mult × learning_rate = 0.08 × 1e-5 = 8e-7、weight decay 0 の独立 param group。
  • anchor KL λ = 0.15 (--router-anchor-weight)、base 256-way pre-top-k router snapshot に対する KL。k 非依存(top-k 選択の前に softmax=分布全体を base に近く保つ、毎 step 1回、二重 forward 不要)。
  • 決定論 env コスト +8.9%：det on 61.96 s/it vs off 56.87 s/it (n=2/腕, ABBA, same-condition, gpu-host GPU0-7, ckpt-5 resume, GA4/seq7168/adafactor)。腕内ばらつき ≤±0.9%。200-step は on 推奨(+約17分)、本走 on/off はユーザー判断(72h 換算 +6.4h)。
  • 決定論設定: CUBLAS_WORKSPACE_CONFIG=:4096:8 + NCCL_ALGO=Ring + NCCL_PROTO=Simple + torch deterministic core + TF32 off。
  • GA4 + sync_each_batch=true(Transformers 5.7 accelerator_config、Full-FFN 限定):各 microbatch で勾配を shard/reduce し full-gradient 常駐を防止(OOM 対策の本丸)。global batch 32 維持。
  • FSDP full-shard 安全化 (07-11 修正): joint anchor が全 rank で vec(0) クラッシュ(gate.weight を forward 外参照)→ Qwen3_5MoeTopKRouter の forward hook で top-k 前 full 256-expert router_logits を捕獲、base gate weight は CPU snapshot に保持、live parameter を後段参照しない。grad-checkpoint 再計算 hook は無視。
  • checkpoint: DCP marker schema 3 に router_joint 設定(enabled / router LR mult / effective LR / anchor weight / stride)を保存。joint resume は同一設定を要求、旧 frozen marker からの joint resume は拒否(frozen→frozen resume は互換)。
  • load-balancing aux loss は joint でも混入しない(--fused-ce/FLCE は backbone hidden から直接構成、gate への補助目的は明示した anchor KL のみ)。

評価層

ローカル 2×RTX PRO 6000(GPU0/1 のみ)— base→候補 直列 paired

  • ツールコール = BFCL v4：非 live / 非 external-API AST、pinned Gorilla + local bfcl-eval wheel、公式 AST parser/checker。
  • コーディング = HumanEval：bubblewrap isolated。
  • 日本語知識 = JMMLU：choice-logprob。
  • 日本語指示追従/一貫性 = M-IFEval日：strict scorer、seed 分散 + seed 間一致率。
  • 一般能力 (非劣化ゲート): MMLU (choice-logprob) / GSM8K。
  • 判定規律: 同一問題・同一 seed の paired + CI95 + McNemar のみ信用。単発・条件違いは参考値。4軸 protocol はまだ未凍結(パイロット段階)。

判定ゲート

4ゲート全通過 (measured / gpu-host 実機) → 200-step probe 発射済み (2026-07-11)

  • ✓OOM 解消 — Full-FFN DCP probe v2 で 8GPU peak max_memory_allocated 63.59 GiB(<70 GiB 目標)。決定論 probe(arm A)でも peak 63.64 GiB。measured, gpu-host 実機(v2=63.59 GiB / 決定論 arm A=63.64 GiB)。v1 は各 GPU 93–94 GiB で全 rank OOM → sync_each_batch で解決。
  • ✓決定論 GREEN (exact-resume) — arm A/B cold/cold、6段階アサート 全 MATCH(load_model / load_optimizer / RNG / batch_loss ×32 microbatch / clip 後 gradient / post_optimizer、各 8 rank)。bit 不一致源 = grouped-mm backward / NCCL reduction の再起動間非決定性で確定、決定論 env で消える → 200-step の技術的ブロッカーなし。
  • ✓grad ゲート (frozen Full-FFN) — expert grad union 80/80、frozen (router/attn/embed) grad は all None。measured, GPU 実測。
  • ✓router-joint grad ゲート (gradgate v3, 2026-07-11) — GPU 実機 8 rank で router union coverage 40/40(全 40 層の gate が勾配受領)、expert union 80/80、attn/embed grad all None、router group LR = 8e-7(0.08 × 1e-5)を実測確認。加えて fresh 6-step vs checkpoint-5 resume の STEP6_POST_OPT digest(model / optimizer_tensors / optimizer_scalars)が全 8 rank で bit 一致。measured, gpu-host 実機 (fresh vs resume, same-condition)。CPU predicate test 4/4 PASS(test_fullffn_joint_trainer.py)。v1=anchor KL の FSDP vec(0) クラッシュ→ forward hook 修正、v2=--skip-final-checkpoint が save-steps ごと無効化する罠→ fresh 段から除去、v3 で PASS。

# qwen36-a6b-toolcall-intent

Machine-verified, intent-level tool-calling SFT data, built for the
[qwen36-a6b](https://github.com/hikarioyama/qwen36-a6b) project — an open
experiment that widens Qwen3.6-35B-A3B from 8 to 32 active experts (k=32,
frozen router, inference-time tail-scale calibration) and then trains the
full FFN to convert the extra active capacity into capability.

Every row is a complete chat episode: a natural-language user request, the
tool schemas available to the assistant, and an assistant response whose tool
calls were **verified mechanically against the episode's expected call
graph** (names, arguments, parallel groups, dependencies, and error-recovery
branches) before admission. Nothing in this dataset was judged "good" by a
model; a row is here because the target model actually solved it and the
solution was checked programmatically.

## Why this dataset exists

An earlier fuel iteration for the same project used mechanically templated
tool names (`mock_finance_165_inspect_1`-style). Training on it silently
destroyed function-name copy fidelity: the trained model began *reconstructing*
names instead of *copying* them from the schema
(`get_fibonacci_sequence` → `get_f fibonacci sequence`), costing 31 points on
a BFCL-style paired evaluation (n=100, McNemar exact p=1.2e-07) while
MMLU/JMMLU/GSM8K moved by nothing. This dataset is the corrected fuel, and its
design choices are all downstream of that failure analysis:

1. **Diverse, realistic tool names.** Five morphologies (snake_case,
   camelCase, dotted namespaces, server-prefixed hyphen forms, short
   verb-noun compounds), tens of thousands of unique names, never a bare
   English word, with numeric suffixes rate-limited. Names collide with
   nothing in the BFCL evaluation set (normalized-name collision guard).
2. **Semantically grounded schemas.** Every tool parameter carries an English
   description; schemas without one are rejected at generation time.
3. **Strict paraphrase gates, no fallback.** User requests are LLM
   paraphrases of formal operation transcriptions, admitted only if they pass
   machine gates: typed literal fidelity (identifiers verbatim, numbers by
   numeric equality), schema-derived name-leak rejection (normalized search
   over all tool and parameter names), operational-invention rejection (no
   numeric or quoted literal outside the episode's value set), batch-level
   opening-bigram and style quotas, and character-3-gram near-duplicate
   removal. Rows that fail any gate are discarded — never backfilled with the
   formal transcription.
4. **Style diversity by construction.** Each seed is assigned one of six
   style cards (neutral/direct, terse mobile, conversational, professional
   email, informal, context-first) with fixed quotas, so the request
   distribution is not one register of polite business English.

## Difficulty tiers

| Tier | Share | Structure |
|------|-------|-----------|
| T2 | 15% | single-stage calls, intent-level phrasing |
| T3 | 20% | multi-stage with distractor tools |
| T4 | 65% | long dependency chains with parallel groups, derived values, and error-recovery branches, plus distractor tools |

T1 (verbatim formal transcriptions) is intentionally absent: under the
no-fallback policy, transcription-register rows are exactly what we do not
want the model to learn from.

## Generation pipeline

1. **Seed synthesis** (procedural): tool schemas, expected call graph, and a
   formal transcription request, with the diverse-name and description
   requirements above enforced at generation time.
2. **Paraphrase** (LLM): the transcription is rewritten as a natural user
   request under the seed's style card, then filtered by the machine gates.
3. **Rejection sampling** (self-generation): the base model
   (Qwen/Qwen3.6-35B-A3B) generates candidate episodes; only episodes whose
   tool calls verify exactly against the expected call graph are kept.
4. **Decontamination**: normalized 8-gram overlap and evaluator function-name
   collision checks against BFCL sources, applied to every row.

## Fields

Each JSONL row contains a `messages` list (system/user/assistant turns with
native `<function=...>` tool-call formatting for the Qwen3.6 chat template),
plus metadata: `seed_id`, `tier`, `style_card`, and `source_tag`.

## Lineage and licensing notes

Paraphrase text was produced with the authors' own assistant subscriptions
(GPT-5.3 lane) and, in earlier rounds, GLM-5.2; assistant-turn text is
self-generated by Qwen3.6-35B-A3B and verified mechanically. Tool names,
schemas, and values are procedurally synthesized and refer to no real API or
service. The dataset contains no personal data and no scraped web text.

## Caveats

- Requests are English-only in this release.
- The verification gate checks call structure, not prose quality of the
  assistant's surrounding text.
- Rows were admitted by rejection sampling against a single base model;
  difficulty is therefore calibrated to Qwen3.6-35B-A3B's frontier
  (T4 admission ≈ 26–28% of attempts) and may transfer differently to other
  models.
